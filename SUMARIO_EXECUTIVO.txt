================================================================================
                              1. DIAGNÓSTICO
================================================================================

PROBLEMA PRINCIPAL:
  App travava em tela "Carregando" por tempo indefinido, especialmente no iPhone.

CAUSA RAIZ:
  Combinação de 10+ problemas simultâneos (não 1 único problema):
  • APIs sem timeout → promises pendurendes infinitamente
  • Modal de ID sem timeout → trava aguardando identificação
  • IndexedDB falhava em modo privado iOS → sem fallback
  • Service Worker frágil → precache all-or-nothing
  • Cache preso no Safari iOS → força a carregar versão antiga
  • Viewport não respeita safe-area → modal coberta por notch
  • localStorage sem suporte → offline com falha
  • DOMContentLoaded registrado 2x → dupla inicialização

IMPACTO:
  🔴 CRÍTICO — app inutilizável no iPhone
  🟠 ALTO — risco em conexões lentas em qualquer device
  🟡 MÉDIO — dev experience ruim (sem logs estruturados)

================================================================================
                            2. SOLUÇÃO IMPLEMENTADA
================================================================================

ESTRATÉGIA:
  Implementar 13 correções críticas que trabalham JUNTAS:

1. TIMEOUTS DEFENSIVOS EM TUDO
   └─ Nenhum await/fetch sem timeout máximo
   └─ APIs: 8-10s | Init: 45s | ID: 30s | Módulos: 5s
   ✅ ANTES: promises infinitas
   ✅ DEPOIS: app abre mesmo com rede lenta

2. GUARDS CONTRA DUPLA INICIALIZAÇÃO
   └─ Flag _isInitializing previne re-entrada
   └─ forceFinishInit() em timeout crítico
   ✅ ANTES: tudo inicializa 2x
   ✅ DEPOIS: init seguro e único

3. FALLBACK PARA INDEXEDDB
   └─ Detecta modo privado iOS automaticamente
   └─ Fallback transparente para localStorage
   ✅ ANTES: app falha em modo privado
   ✅ DEPOIS: funciona em qualquer modo

4. SERVICE WORKER ROBUSTO
   └─ Precache item-by-item com fallback
   └─ Continua mesmo se alguns assets falham
   └─ Timeouts em fetches do SW
   ✅ ANTES: precache all-or-nothing falha
   ✅ DEPOIS: fallback offline sempre funciona

5. LIMPEZA EMERGENCIAL PARA iOS
   └─ Auto-dispara se travamento detectado
   └─ Limpa caches + localStorage + recarrega
   ✅ ANTES: cache preso força reload manual
   ✅ DEPOIS: auto-limpa silenciosamente

6. VIEWPORT E SAFE-AREA PARA iOS
   └─ CSS com env(safe-area-inset-*)
   └─ Modal respeita notch e home indicator
   └─ Teclado virtual não quebra layout
   ✅ ANTES: modal coberta/inacessível
   ✅ DEPOIS: UI perfeita no iPhone

7. PROTEÇÃO DO DOMCONTENTLOADED
   └─ Guard _domContentLoadedFired
   └─ Previne disparo duplicado
   ✅ ANTES: pode disparar 2x
   ✅ DEPOIS: always single dispatch

8. DETECÇÃO DE iOS/SAFARI
   └─ Flag global window._APP_PLATFORM
   └─ Usado para lógica específica iOS
   ✅ ANTES: sem forma de detectar iOS
   ✅ DEPOIS: code paths otimizados por device

9. FALLBACK LOCALSTORAGE
   └─ Se IDB falha, automático usa localStorage
   └─ Keys com timestamp + random
   └─ Suporte completo a offline
   ✅ ANTES: offline com falha IDB = perda de dados
   ✅ DEPOIS: sempre persiste mesmo em modo privado

10. LOGS ESTRUTURADOS
    └─ Prefixes: [INIT], [API], [DB], [SW], [CLEANUP]
    └─ Emojis para fácil visualização
    └─ Fácil encontrar problema no console
    ✅ ANTES: logs espalhados sem padrão
    ✅ DEPOIS: debug rápido e estruturado

11. PROTEÇÃO MODULAR
    └─ Try/catch em cada inicialização
    └─ Módulos não-críticos não param init
    ✅ ANTES: 1 erro para todo o app
    ✅ DEPOIS: app abre mesmo com erro em módulo

12. ASYNC/AWAIT SEGURO
    └─ Helper withTimeout() para todos os awaits
    └─ Finally obrigatório para desligar loader
    ✅ ANTES: loader pode ficar preso
    ✅ DEPOIS: loader SEMPRE desliga

13. COMPATIBILIDADE BACKWARD
    └─ Zero breaking changes
    └─ Tudo é additive (só melhora)
    ✅ ANTES: risco de quebrar funcionalidades
    ✅ DEPOIS: 100% compatível

                           7. COMO TESTAR RÁPIDO
================================================================================

DESKTOP:
  1. Abrir em Chrome (F12 → Console)
  2. Verificar logs [INIT], [API], [SW]
  3. Confirmar que "Carregando" desaparece em ~5s

ANDROID:
  1. Abrir em Chrome Mobile
  2. DevTools via chrome://inspect (no PC)
  3. Verificar responsividade
  4. Testar offline (airplane mode)

iPhone (CRÍTICO):
  1. Abrir no Safari
  2. Deixar por 30s — NÃO deve travar
  3. Modal de ID deve funcionar
  4. Identificar usuário
  5. Fechar e reabrir — deve lembrar do usuário
  6. Testar offline (airplane mode)

Rede Lenta:
  1. DevTools → Network → Slow 3G
  2. Recarregar
  3. Confirmar que espera ~10s mas abre (não trava)

Modo Privado iPhone:
  1. Safari → Abas Privadas
  2. Abrir app
  3. Não deve falhar (usa localStorage)
